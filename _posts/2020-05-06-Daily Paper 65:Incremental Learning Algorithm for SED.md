# Daily Paper 65: Incremental Learning Algorithm for SED  

## Introduction  

这篇文章是UCSD音乐系的学生和高通的几个人发表在ICME2020上的，现在学音乐的都能发CCF B区文章就离谱。这篇文章有意思的地方在于他使用了最近很火的incremental learning增量学习的方式来进行sound event detection(SED)任务。可以学习借鉴一下增量学习在这类任务的应用方式。  

增量学习是一种新的训练方式，该方式可以解决SED的两个问题：从预训练模型到新模型的知识迁移，以及在不忘记之前的学习内容的前提下学习新的声音事件。为了从源模型中将之前学习的知识移植到目标模型中，就需要在源模型的顶部部署一个neural adapter。这个neural adapter layer利用目标模型和最小的数据去学习sound events，从而在维持用之前类似模型学习到的表现的同时力求进一步提升准确率。所谓的增量学习听起来和迁移学习有很多类似的地方，都是通过两个模型来学习一个任务，不过在我的理解中，迁移学习只是利用之前的数据集在新的数据集上力求得到更好的性能，而增量学习的首要目的是解决灾难性遗忘问题，也就是说在学习新知识的时候不能忘记旧知识，对于算法来说之前的模型学习的结果和新模型学习的结果都是同样重要的。  

作者提到了SED任务目前面临的问题，作者认为目前最大的麻烦是SED任务的训练数据类别较少，虽然按照这些数量较小的类别进行训练和测试在数据集上的表现不错，但是如果要在现实中进行落地应用，那么就会存在很多未标注的类别无法识别的问题。但是问题就在于SED任务可能会不断的增加一些当时未发现或者未识别的类别，那么算法就需要不断的增加新的类别进行训练，这种在训练过程中加类别的需求就无法通过迁移学习来完成，只能通过continuous learning持续学习和incremental learning增量学习来完成了。不过我认为lifelong learning好像也能达到这种效果。  

那么这些持续学习最容易面临的问题就是灾难性遗忘，这和人其实差不多，学了后面忘了前面，复习完前面又忘了后面。而这在机器学习中表现的更为明显，因为你一旦用新的模型、新的数据集进行训练后，之前的权重就没有了意义，那么之前学习的信息就会被重写。很多学者也尝试用迁移学习的方式来解决这个问题，但是就目前来说效果还不是很好，所以作者尝试使用增量学习的方式来解决这一问题，并取得了超越其他迁移学习方式的效果。  

## Method  

首先介绍一下SED任务，SED任务的输入是一段音频，算法需要标注出这一段音频中的所有声学事件和起始时间点。整体的流程图如下图所示:  
![65-1](/images/daily paper/65-1.png)  

这个图画的实在是拉跨，据我理解，作者的增量是把原来的三类别增加为四类别，也就是增添一个类别。最左侧上面那个是source model，下面那个是target model，首先用source data对source model进行训练，得到source output，然后将source model的权重transfer到target model中，其实就是个权重迁移，然后用target data，也就是四类别的数据集训练target model。到这里流程和普通的迁移学习是一模一样的，而区别在于作者将Neural Adapter的结果和Target Model的结果通过一个Merger结合在了一起，再得到最后的target output。增添的模块有两个：Merger模块和Neural Adapter模块。  

对于Merger模块来说，内部的东西其实很简单，就是逐元素相加然后过一个sigmoid保持结果的范围，而对于Neural Adapter，这里是通过了两个FC层，第一个FC层是100维的，第二个是4维的。到这里整个算法的流程就结束了，看完我的内心是崩溃的，所谓的增量学习模型，就是多了两个FC层然后把结果加起来，就这个两个FC层的Neural Adapter还是借鉴的人家NLP一篇论文的算法。。。  

## Experiment  

结果我都不想截图了，作者的baseline比较根本就没有意义，加一个Neural Adapter和不加的迁移学习模型进行对比，这根本不是baseline，而更像是一个消融实验，加Neural Adapter的表现没有理由比不加的差，没有和其他类似算法的横向比较完全没有意义。  

## Conclusion  

不知道是不是因为我一直看的都是顶会的文章，对于ICASSP的文章还算能看得下去，但是ICME的这篇文章我根本不知道发出来的意义是什么，作者做了详细的ablation study，在三个数据集上翻来覆去的比较Neural Adapter的作用，但是这些我认为大部分都是废话。如果要说明效果好，至少要和其他的增量学习，或者至少是迁移学习方式进行对比，但是很遗憾没有对比，所以看不出效果如何。此外，这篇文章的创新点就是把NLP的一篇neural adapter的思想搬到了sound event detection上，然后拿着incremental learning的噱头发文章，但是实质性的改进只有两个FC层。我也明白很多出名的网络改进都是极小的修改，比如ResNeXt等一系列网络，但是他们都至少在理论层面证明了他们的改进是有依据的。而这篇文章既看不到理论证明、又看不到架构的创新、更看不到最后的结果，我看了半天感觉除了浪费时间啥都没学到，只知道以后还是尽量避开ICME的文章看吧。
